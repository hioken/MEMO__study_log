# 概要
- 事前にコンパイルする
- 独自の記述を追加し、コンパイラに厳重にチェックさせる
- コンパイル時に、tsの記述は削除される
- 例外的に変換されるものもある
  - enum
  - namespace
  - コンストラクタの引数プロパティ

# TSコンパイラ
## 概要知識
### CFG(コントロールフローグラフ)
- ASTを元にCFGを作成
- このCFGはフローの流れに加えて、変数の型の保証も行う
  - `if (!data) return 0;`このコード以下では、`data`は再定義/代入されるまで真の値である事が保証される

## コンパイルプロセス
## 1. Parser（構文解析）
- ソースコードの文字列を字句解析・構文解析し、AST（抽象構文木）をメモリ上に構築する
- JS本来の値のノードと、TS固有の型のノードを混在させた状態のツリーを生成する

## 2. Binder（Symbolリスト作成）
- 構築されたASTを走査し、識別子（変数等）とスコープ・処理位置情報を紐付けた「Symbolリスト」を作成する
- コード全体の「どこに何が宣言されているか」の静的なカタログ化を行う

## 3. Type Checker（矛盾検知）
- ASTとSymbolリストを入力として受け取り、CFG（コントロールフローグラフ）を構築する
- 条件分岐や早期リターン等の処理の切れ目を考慮して、各フローにおける型の状態遷移（型の保証）を追跡する
- Symbolリストに登録された型定義と、CFG上の実際の利用パスを突き合わせ、論理的な矛盾（Type Error）を検知する

## 4. Emitter（型の消去と出力）
- 型検査の完了後、ASTから型のノードおよび型のSymbolを物理的にすべて削除（Type Erasure）する
- 残された値のノード群のみをテキストに変換し、ピュアなJavaScriptファイルとして出力する
